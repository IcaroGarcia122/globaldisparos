# WhatsApp Adapter Migration Guide

## Overview

The application now supports multiple WhatsApp integration methods through an **adapter pattern**. This allows seamless switching between different implementations without changing the rest of the codebase.

### Supported Adapters

1. **Baileys Adapter** (Default) - Direct WebSocket connection
   - Uses `@whiskeysockets/baileys` library
   - Maintains persistent connection to WhatsApp Web
   - Stored auth credentials in `backend/auth_sessions/`
   
2. **Evolution API Adapter** (New) - REST API integration
   - Uses Evolution API (HTTP REST)
   - Webhook-based event handling
   - No local credential storage
   - Requires Evolution API server running

---

## Quick Start - Switching Adapters

### Using Baileys (Default)

```bash
# Set environment variable
export WHATSAPP_ADAPTER=baileys

# Or in .env file
WHATSAPP_ADAPTER=baileys

# Start server
npm run dev
```

### Using Evolution API

```bash
# Copy environment template
cp .env.evolution .env

# Update .env with your Evolution API details
WHATSAPP_ADAPTER=evolution
EVOLUTION_API_URL=http://localhost:8080
EVOLUTION_API_KEY=your-api-key

# Start server
npm run dev
```

### Component Hierarchy

- **WhatsAppAdapter** (Abstract Base)
  - Defines interface/contract
  - In: `src/adapters/WhatsAppAdapter.ts`

- **BaileysAdapter** (Implementation 1)
  - Encapsulates Baileys logic
  - In: `src/adapters/BaileysAdapter.ts`
  - Replaces: `src/services/baileysService.ts`

- **EvolutionAdapter** (Implementation 2)
  - Evolution API implementation
  - In: `src/adapters/EvolutionAdapter.ts`

- **WhatsAppService** (Facade)
  - Delegates to active adapter
  - In: `src/adapters/WhatsAppService.ts`

- **whatsapp.config** (Factory)
  - Creates WhatsAppService with correct adapter
  - In: `src/adapters/whatsapp.config.ts`

## Backward Compatibility

✅ **Zero Breaking Changes**

- All existing endpoints work unchanged
- Database schema is unchanged
- Frontend code needs no modifications
- Environment variable defaults to `baileys` (current behavior)

### Migration Path

**Phase 1: Code Deployment** (Now)
- Deploy new adapter infrastructure
- Default is still `WHATSAPP_ADAPTER=baileys`
- Production uses existing Baileys connection

**Phase 2: Testing** (When ready)
- Switch test environment to Evolution
- Verify all features work with new adapter
- Test webhook event handling

**Phase 3: Gradual Rollout** (Optional)
- Create new tenant instances with Evolution
- Keep existing instances on Baileys
- Switch instances gradually as needed

**Phase 4: Full Migration** (Optional)
- Migrate all instances to Evolution
- Retire Baileys code if desired


| Variable | Description | Example |

## Implementation Details

### Baileys Adapter

```typescript
// Uses existing baileysService code
// - Manages WebSocket connections
// - Stores credentials in auth_sessions/
// - Listens for events via socket.ev.on()
// - No HTTP calls needed

import baileysService from './adapters/BaileysAdapter';
```

### Evolution Adapter

```typescript
// New HTTP-based implementation
// - Makes REST calls to Evolution API
// - No persistent socket needed
// - Receives events via webhooks
// - Configured via environment variables

import evolutionAdapter from './adapters/EvolutionAdapter';
```

### Adding a New Adapter

1. Create new file: `src/adapters/NewAdapter.ts`
2. Extend `WhatsAppAdapter` abstract class
3. Implement all abstract methods
4. Register in `src/adapters/whatsapp.config.ts`:

```typescript
case 'newadapter':
  return new WhatsAppService(NewAdapter);
```

## Troubleshooting

### Evolution API Connection Issues

```
Error: EVOLUTION_API_KEY environment variable is not set
```

**Solution:** Add `EVOLUTION_API_KEY` to `.env`

```
Error: Cannot POST /message/sendText
```

**Solution:** Verify `EVOLUTION_API_URL` is correct and Evolution API is running

### Switching Between Adapters

```bash
# Stop server
# Update .env
WHATSAPP_ADAPTER=evolution

# Restart server
npm run dev
```

### Instance Not Connecting

**Baileys:** Check `backend/auth_sessions/` directory permissions  
**Evolution:** Check Evolution API health: `GET EVOLUTION_API_URL/health`

## FAQ

**Q: Do I need to change my frontend code?**  
A: No, all endpoints are unchanged.

**Q: Can I use both adapters at the same time?**  
A: Not per instance, but you can switch globally with environment variable.

**Q: Will my instances auto-migrate?**  
A: No, you must explicitly configure which adapter to use.

**Q: What happens to my auth credentials?**  
A: Baileys: stored locally as before. Evolution: managed by Evolution API.

**Q: Can I switch back to Baileys?**  
A: Yes, just change `WHATSAPP_ADAPTER=baileys` and restart.
